Before you start

Reclaim scans your files and suggests things to delete — it never deletes anything without your review.

  • Safe mode is on by default. Everything is recommend/review-only: you pick what to remove, nothing is applied automatically.
  • Deletes go to the Recycle Bin in safe mode — never a permanent delete. Restore anything from Windows' own Recycle Bin at any time, the same as if you'd deleted it yourself.
  • Some categories stay off by default (exact-duplicate detection, ML model caches, dev-environment folders) — they reach into places a false positive would hurt most.
  • Power mode unlocks the full toolset (permanent delete for rebuildable caches, auto-apply) — it's opt-in, requires typing a confirmation phrase, and you can switch back to safe mode at any time.
  • Residual risk, stated honestly: environments and project layouts aren't universally modeled. Review what's proposed before confirming, especially anything outside the categories above.

    How this works

    • Deterministic rules do the safe cleaning. Every category shown here (developer build artifacts, package caches, temp/browser caches, crash reports, and similar) is detected by fixed, inspectable rules — never a probability score — and every rule states the mechanism that makes it safe (e.g. "rebuilds on next npm install").
    • AI only ever suggests, never deletes. Anything an AI feature surfaces (similar photos, possible document drafts, screenshot bursts) is a recommendation you review — it can never reach the apply step on its own.
    • In safe mode, everything goes to the Recycle Bin. Nothing is permanently deleted by default. Restore anything from Windows' own Recycle Bin at any time, the same as if you'd deleted it yourself — see the Quarantine & Restore tab for a list of what's been moved.
    • Power mode (opt-in, typed confirmation required) unlocks permanent delete for rebuildable caches and the full toolset — you can switch back to safe mode at any time with no confirmation.
